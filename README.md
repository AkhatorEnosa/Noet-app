
# Noet – WroteNote(Official name)

Modern, seamless note-taking web application.

Built with **React** + **Vite**, styled with **Tailwind CSS**, animated with **Framer Motion**, using **Supabase** for authentication & persistent storage, **Redux Toolkit** + **Thunk** for state management, and **TanStack React Query** for efficient data fetching & synchronization.

🌐 **Live Demo**: https://wrotenote.netlify.app

## ✨ Features

- Secure email/password authentication (Supabase Auth)
- Create, read, update, delete personal notes
- Full-text search
- Note sharing
- Note privacy
- Smooth UI transitions and micro-animations
- Responsive design across devices
- Fast development & hot-reloading (Vite)
- Efficient server-state handling & caching
- Clean, maintainable architecture

### Planned / Coming Soon

- Note categories, tags or folders
- Dark / light mode toggle
- Markdown editor + preview
- Offline support (IndexedDB + sync when online)
- Note export / import
- PWA (installable app)

## 🛠 Tech Stack

- **Frontend**: React 18, Vite
- **Styling**: Tailwind CSS
- **Animations**: Framer Motion
- **State Management**: Redux Toolkit + Redux Thunk
- **Data Fetching & Sync**: @tanstack/react-query
- **Backend & Auth**: Supabase (PostgreSQL + Auth)
- **Deployment**: Netlify
- **Linting**: ESLint

## 🚀 Quick Start (Local Development)

### Prerequisites

- Node.js ≥ 18
- Supabase account & project (free tier works)

### Steps

1. Clone the repository

   ```bash
   git clone https://github.com/AkhatorEnosa/Noet-app.git
   cd Noet-app
   ```

2. Install dependencies

   ```bash
   npm install
   # or
   pnpm install
   # or
   yarn install
   ```

3. Create `.env` file in root with your Supabase keys:

   ```env
   VITE_SUPABASE_URL=https://your-project-ref.supabase.co
   VITE_SUPABASE_ANON_KEY=your-anon-public-key
   ```

   → Find these in Supabase → Settings → API

4. Start development server

   ```bash
   npm run dev
   ```

   Open http://localhost:5173 (or the port shown)

### Other Commands

```bash
npm run build     # production build → dist/
npm run preview   # preview production build locally
npm run lint      # run ESLint
```

## Supabase Setup (One-time – required for the app to work)

1. Create project → https://supabase.com/dashboard
2. Go to **SQL Editor** and run:

   ```sql
      -- Create the notes table (your exact schema)
      CREATE TABLE public.notes (
        id          BIGINT GENERATED BY DEFAULT AS IDENTITY NOT NULL,
        created_at  TIMESTAMPTZ NOT NULL DEFAULT NOW(),
        data_value  VARCHAR NULL,                    -- main note content / body
        bg_color    TEXT NULL DEFAULT '',
        img         TEXT NULL DEFAULT '',
        index_num   BIGINT NULL,
        user_id     UUID NULL,                       -- links to auth.users
        pinned      BOOLEAN NULL DEFAULT FALSE,
        privacy     BOOLEAN NULL DEFAULT TRUE,
        title       TEXT NULL,
        CONSTRAINT  data_pkey PRIMARY KEY (id)
      ) TABLESPACE pg_default;
      
      -- Enable Row Level Security (very important for multi-user safety)
      ALTER TABLE public.notes ENABLE ROW LEVEL SECURITY;
      
      -- Policy: Users can only SELECT their own notes
      CREATE POLICY "Users can view their own notes"
        ON public.notes
        FOR SELECT
        USING (auth.uid() = user_id);
      
      -- Policy: Users can only INSERT notes for themselves
      CREATE POLICY "Users can insert their own notes"
        ON public.notes
        FOR INSERT
        WITH CHECK (auth.uid() = user_id);
      
      -- Policy: Users can only UPDATE their own notes
      CREATE POLICY "Users can update their own notes"
        ON public.notes
        FOR UPDATE
        USING (auth.uid() = user_id)
        WITH CHECK (auth.uid() = user_id);
      
      -- Policy: Users can only DELETE their own notes
      CREATE POLICY "Users can delete their own notes"
        ON public.notes
        FOR DELETE
        USING (auth.uid() = user_id);
   ```

3. (Optional) Enable Realtime on the `notes` table for future live sync.

#### Authentication

* Go to **Authentication > Providers**.
* Ensure **Email/Password** is enabled.
* (Optional) Enable **Google OAuth** if you want to use the social login feature.

## Deployment (Netlify)

- Connect to GitHub
- Environment variables set: `VITE_SUPABASE_URL` and `VITE_SUPABASE_ANON_KEY`
- Auto-deploys on push to main

To deploy your own version:
1. Fork or clone repo
2. Connect to Netlify
3. Add the two environment variables above
4. Deploy
